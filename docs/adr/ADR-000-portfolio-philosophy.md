# ADR-000: Portfolio Philosophy

## Status

Accepted

---

## Context

The objective of this portfolio is to demonstrate the skills required for modern AI Platform Engineering roles rather than collecting unrelated technology demonstrations.

---

## Decision

Build a small number of interconnected, production-quality repositories.

Each project must:

- Solve a real engineering problem
- Be reproducible
- Include architecture documentation
- Include automation
- Include testing
- Be deployable

---

## Consequences

Pros

- Strong interview discussions
- Cohesive portfolio
- Easier maintenance

Cons

- Fewer total repositories
- Longer development time
